# Repositório Git

## Informações do Repositório

O projeto está versionado no Git e disponível para revisão pelos tutores e professores.

### Estrutura do Repositório

```
cupcakes-cs/
├── backend/          # API NestJS
├── frontend/         # Aplicação Next.js
├── docs/             # Documentação do projeto
│   ├── situacao-1/  # Documentação Situação 1
│   ├── situacao-2/  # Documentação Situação 2
│   └── situacao-3/  # Documentação Situação 3
├── README.md         # Documentação principal
└── .gitignore        # Arquivos ignorados pelo Git
```

## Branches

- **main**: Branch principal com código de produção
- **develop**: Branch de desenvolvimento (se aplicável)

## Commits

Os commits seguem o padrão de mensagens descritivas:
- `feat: adiciona funcionalidade X`
- `fix: corrige bug Y`
- `docs: atualiza documentação`
- `refactor: refatora código`

## Histórico de Desenvolvimento

O repositório mantém todo o histórico de desenvolvimento do projeto, incluindo:
- Evolução do código
- Decisões de arquitetura
- Correções de bugs
- Implementação de features

## Acesso

O repositório está disponível para:
- Tutores e professores do curso
- Revisão de código
- Demonstração de experiência técnica
- Portfólio profissional

## Continuidade

Este repositório será mantido e atualizado mesmo após a entrega do PIT II, servindo como:
- Portfólio permanente
- Demonstração de habilidades técnicas
- Base para futuros projetos
- Referência de experiência
